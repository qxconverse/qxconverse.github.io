论学习与拖延的关系

Posted on 2019-04-30 | In 思维

这是学习相关的表：

时期 知识来源 检验形式 检验周期 督促人
大学之前 书本 考试 每天 老师（强）、父母（强）
大学 书本、书外 考试、项目 每周 老师（弱）
研究生 书外 考试、论文 每月 导师（弱）
社会上 没有书 项目、处事 看情况 无

可以发现，从小到大，书本上的东西越来越少，书本外的越来越多；而检验形式、检验周期也越来越宽松，从大学之前的每天检查，到大学时候的每周（因为课很少，同样的课，基本上一周一次）。再到研究生，没有课了，但是检查周期也变为月。到了社会上（可能会夹杂在大学及研究生中），基本上没有固定的期限，比如一个项目可能是几个月，几年等等不限。工作中，一个功能上线可能几周，几个月等等。

检验形式、期限的宽松会带来标准的不固定，多了很多主观因素和人事因素，所以更看重综合实力。但无论如何，做好了之后才有资格到主观因素。

同时，标准的不固定，也导致了拖延。大学之前，老师布置作业，第二天会检查，所以我必须今天之内做好。大学时不一样了，给你一周的时间，这时，就会拖延，到最后几天才开始赶，赶不了的（或太困难的）就放弃。研究生时候更严重，比如老板之前安排的论文任务，由于很复杂，所以真的会拖延一个月。到了社会上，比如学车，我tm居然拖了3年。恋爱，拖了7年。

你看，为什么会导致这样的问题，我知道和我不勇敢、不主动有很大关系。但其中有一些理论在里面的。

仔细分析，为什么大学之前我学习很好，不是因为我很聪明，而是因为我把90%以上的时间放在了学习上，我很听话，老师家长说要好好学习，我就好好学习了。大学时，除了学习，还有其他的事情，所以我花费在学习考试的时间上大大减少，又因为检验周期变长，老师基本上不管，于是，学习成绩不如以前，而且各种项目也完成的不好。研究生就更加可怕了，连课都没有了，导师布置一个任务，比如论文，周期往往是一学期，甚至1年，导师基本上不管。社会上，则没有人管了，一切靠你自己以及人脉。

综合，问题的原因在于

  1. 知识的来源越来越多
  2. 事情变多
  3. 检查形式变宽松
  4. 检查周期变长
  5. 督促自己的人越来越少

这些都导致了自己很多事情拖延，或做不好。

解决办法：

  1. 不要只从书本中获取知识（不单单是书本），而要广泛地获取，社会上的人情世事也是知识。
  2. 安排好各种事情的时间，一件一件来。
  3. 自己给自己设定很严格的标准，比如让自己一天内做完，比如让自己做到什么程度。给自己设置以天为单位的deadline。虽然事情很大，一天完成不了，但可以每天给自己设定小任务。
  4. 自己给自己暗示，做自己的督促人，这就要求自律。

3-24

第一讲：密码学基础

区块链中密码学的核心概念：hash和签名

hash，它有三个重要的特性：collision resistance, hiding, puzzle friendly，目前用SHA-256（secure hash algorithm）

碰撞抵制：很难找到x,y使得h(x)=h(y)
hiding: x->h(x)容易，但是h(x)->x难
puzzle friendly：其分布难以控制

比如sealed envelope，A方发布消息，B方接受消息。hiding可以保证B方很难破解消息，而碰撞抵制可以防止A方进行篡改。这个签名过的消息则保存在可信的第三方。

在密码学中，分为对称加密与非对称加密。非对称加密中，包含有公钥与私钥(public key, private key)。pk加密，sk解密。需要注意的是用的是同一人的pk和sk。而签名正好相反，sk签名，pk验证。

区块链中，发送消息，通常是先hash，然后再签名。

3-25

第二讲：数据结构

比特币中的一个数据结构是区块链

重要概念：hash指针，除了包含地址，还包含结构体的hash值（为的是防篡改）

区块链是使用了hash指针的链表（linked list）

某个区块里面的hash指针的取值为，前面整个区块的内容和它里面的指针一起hash得到的结果。

最后一个保存在系统中的hash指针值可以检测出前面任意的区块是否被篡改，叫做tamper-evident log。牵一发动全身，多米诺骨牌的效果。

比特币中的另外一个数据结构叫做merkle tree。

最底层为data block，其它的节点都是hash指针，对根节点节点取的hash值为root hash。与区块链类似，root hash可以检测对其它任意内容的篡改，而且效率更高，因为其类似于二叉树的结构。

每个区块所包含的交易会组织成merkle tree，最底下的数据块都是比特币的交易块。

每个区块分为块头和块身，block header和block body。block header只记录root hash，而具体的交易记录在block body中。

merkle proof，是一连串的merkle路径，其可以证明merkle tree中包含了某个交易。（复杂度logn）

sorted merkle tree（对交易块取hash值，并排序），用于不存在证明（这样的复杂度还是logn）。比特币中因为不需要不存在证明，所以不是用排序merkle tree

比特币中的分叉

fork

state fork 有分歧而产生的分叉

    forking attack

    deliberate fork

protocol fork 比特币协议发生了变化

    hard fork

        block size limit（如果每个区块1M大小，那么会有4000个交易，而10分钟产生一个交易，所以平均每秒产生7个交易）比如1M->2M（必须所有的节点都要发叉）。新节点认为合法的，旧节点不认为合法的；旧节点认为合法的，新节点也认为合法的。

    soft fork

        对协议加一些限制，使得在新的协议中不再合法，比如1M->0.5M，属于临时性的分叉。旧节点认为合法，新节点认为非法的。反之，新节点认为合法的，旧节点也认为合法的，所以旧节点会自动合并进去。旧节点如果不更新，会挖很多无用的块（浪费时间）。还有一个例子，出现了coinbase域，可以作为extra nonce（不止8字节）。本来nonce只有4bytes。手机上app里面的基本上都是轻节点。UTXO组成一个merkle tree，其根hash值放入到coinbase。（只要拥有半数以上算力节点更新了，就不会出现永久性分叉）

判断是软硬分叉，核心看新和旧相互是否认可为合法的。

P2SH（pay to script hash，后来加进去的）属于软分叉

GNP

GNP(国民生产总值)与GDP(国内生产总值)都是作为国民收入核算的指标,现在多数国家较多的使用GDP指标 GNP是与所谓国民原则联系在一起,GDP是与所谓国土原则联系在一起 GNP与GDP的关系: GNP=GDP+对外要素收入净额 对外要素收入净额表示本国生产要素在其他国家获得的收入减去本国付给外国生产要素在本国获得的收入

经济与货币政策有很大干系，最终可以维持低通胀

银行产生了很多坏账，银行开始以低首付率贷款，他们先以低利率进入，之后贷款利息一路飙升或进行调整，调整后，贷款人无法承受大额尾付金额。还有，那些卷入次贷的投资和金融机构，银行发放资产抵押债券，转手出卖，接盘者将债券打包后，出售给纽约的大型金融机构，后者将重组后的债券再次出售

泰勒规则：利率应该根据经济形势进行调整，若经济增长缓慢，则降低利率，同时应该考虑通货膨胀的因素；倘若仍存在通货膨胀，应当适当调高利率。

降低：税率，边际税率

经济不景气时，公司的投资支出大量减少，因为他们看不到投资收益，公司不愿意扩大规模来增加产能，而是退守自保，削减设备支出。

恒定的税率改制绝对划算。

自由贸易

国际收支（贸易）逆差，进口商品和服务总值与出口商品和服务总值的差额

其也衡量本国的储蓄和外国在本国的投资，外国投资额大于本国储蓄额时，出现了贸易逆差。

不要过分注重贸易逆差，而忽略自由贸易带来的成效，比如生产力的发展，收入水平的提高。

美元的贬值，刚好可以扭转贸易逆差

维持低通胀，可以使得美元更加坚挺

财政支出与税率

增值税指的是以商品生产、流通和加工、修理等各环节所增加的价值额或商品附加值为对象所征收的一种流转税。

货币+信用=支出总额

支出总额/产销总量=价格

货币、信用——>商品、服务、金融资产

中央政府：收税、花钱
中央银行：控制经济中的货币和信贷数量（通过影响利率和发行更多货币）

信贷是经济中最重要的部分

信贷一旦产生，就会出现债务，债务是借款人的负债，是贷款人的资产。

由于我们借债，于是产生了周期，这是由于人的天性使然

信贷会导致周期性变化

短期债务周期

信贷是否易于获得觉得经济是否扩张，当信贷易于获得时，经济扩张，价格上涨，出现通货膨胀；当信贷不易获得时，经济萎靡，价格下跌，出现通货紧缩。

这一切由央行通过利率控制。短期债务周期大概持续5-8年。

长期债务周期

债务/收入=债务负担

削减支出-收入减少-财富减少-信贷减少-减少借债-削减支出-….

1929,2008但是利率接近于0

以下四种措施用于去杠杆化过程：

个人，企业，政府削减支出，会导致通货紧缩，企业减少成本，工作机会减少，失业率上升

通过债务违约和重组，减少债务，导致萧条

财富再分配，富人->穷人，导致通货紧缩

央行发行货币

财政赤字，即政府的支出超过收入（税收）

从富人增税

发行货币，会导致通货膨胀，刺激经济增长

当找到了通货膨胀和通货紧缩之间找到平衡，就可以和谐地去杠杆化。

收入一定要比债务增长的快

杠杆率可以从侧面反映还款的能力。（各种粒度，小到个人，达到一个国家）

杠杆率与资本金、负债、总资产这三个概念相关。

  1. 总资产=资本金+负债。
  2. 杠杆率=资本金/总资产=资本金/（资本金+负债）
  3. 杠杆倍数=总资产/资本金=（资本金+负债）/资本金

杠杆率的提高，说明了贷款能力的提高。

证件照的制作

Posted on 2019-03-27 | In 人文艺术

如何简单地进行证件照的制作？

  1. 使用单反拍摄原照（最好带三脚架），选择好合适的背景颜色（比如白色）
  2. 将图像传到电脑，用ps打开裁剪，如果是一寸，则选择比例为5：7（准确大小为2.5 × 3.5 cm）；如果是二寸，则比例按照（3.5 × 5.3cm）的标准，分辨率选择300像素/英寸。注意点：如果不需要准确的大小，按照比例裁剪就可以，这时图像大一些，打印出来清晰点。
  3. 如果照片背景不是适合的，则使用智能套索工具选择背景，并替换为合适的颜色。
  4. 如果照片需要美白，则先选择command键不放同时点RGB通道，得到RGB通道高光选区，接着新建一个图层，在图层填充白色，如果发现皮肤过于太白，将图层的不透明度调节到合适的。具体教程参考：https://jingyan.baidu.com/article/48206aeae60025216bd6b35d.html。
  5. 如果不想自己ps修图（比如痘痘、斑纹等细节），可以把照片传入手机，用强大的美图软件搞定。
  6. 将最终的照片打印出来即可，可以买一个小型专业的打印机，调到证件照模式。

狼人杀经验

Posted on 2019-01-06 | In 游戏
  1. 狼队旱跳预言家，给任何人发金水都是毫无意义的。
  2. 如果有2个以上的预言家竞选，并且好人队很难分清，狼人更倾向于给假的预言家投票。
  3. 如果上警的人发言很没自信，那很大程度上不是预言家。
  4. 如果有人穿自己的衣服，就要说出来，虽然不能决定那个产衣服的人的身份。
  5. 如果你是狼人，你穿衣服被发现了，那就主动让出
  6. 分析票型时，要记住谁站自己一边
  7. 给尸体发金水，很有可能是假的预言家
  8. 平民跳了预言家，如果被预言家怀疑了，就要退水
  9. 女巫如果第一晚不救人，那么预言家最多可以验3个人，第一晚一个，第二晚有人杀他，女巫救，第三晚再验一个。同时保持警徽流，即验谁，如果是好人，给他警徽，否则他就是狼人。所以预言家上警很有必要！
  10. 猎人的衣服，一般没有人敢穿，否则作死。但是穿女巫的衣服比较容易，因为你说自己是女巫，事情都是发生在夜晚。没人知道你是不是真的。
  11. 猎人互殴，那是猎人的事情，不用管，女巫也是。真女巫晚上可以毒死假的女巫。如果有人认女巫，那么真女巫就去毒，毒对的概率50%。
  12. 如果自己真的是好人警长，别人也怀疑自己，这时就要分析狼人的数目，表示我如果是狼的话，已经赢了，但是游戏没有结束，说明我不是狼。
  13. 如果你是平民，且发现神已经死得差不多了，这时候要主动穿衣服。那个真的神要去接，表示我真的不是那个神。不然好人方就很难赢了。
  14. 狼人给金水更容易些，因为他能给对，而好人，除了预言家可以给对，其他人都不敢随意给金水的。

从最开始的上警开始：
一般情况下，警上会有狼，也有没有狼的情况，但是多于两狼的情况比较小。

平民一般会有上警的，通常情况下是防预言家对跳；当然，狼人这个时候也有可能上警自称防对跳。比较难判断，当后面有比较可信的强神后，防对跳的基本上全会退水。

前置位平民一般不会跳预言家，因为他也不知道谁是查杀或是金水；因此，前置位跳预言家的一般是狼人或真的预言家。

狼人跳预言家发金水并没有卵用，一般也不会查杀好人，不然会受到好人怀疑；于是常见的玩法是，狼人跳预言家查杀狼人，玩配合。然后被查杀的狼人自曝，以使好人相信假预言家。

如果狼人跳预言家后被怀疑，一般会选择自刀。

女巫的药很关键，开头一般不会用，解药一般会保自己或是救预言家，而毒药一般会在自己死之前撒出去给嫌疑最大的人。

当一个人各种分析晚上毒杀情况，又不是女巫的时候，他很有可能就是狼人，因为他自己最清楚自己杀了谁，而女巫又救了谁或是毒了谁。

当一个人前面竞选警长而又不自信的情况时，很有可能是狼人。

当女巫毒用过了，狼人就可以穿女巫的衣服了。

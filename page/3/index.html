参考博客：http://www.cnblogs.com/hyddd/archive/2009/04/09/1432744.html

关于csrftoken，这篇博客很棒：https://github.com/pillarjs/understanding-csrf/blob/master/README.md

通常的防御措施包括

  1. 只使用json api，这样只能用ajax来提交json，而不能直接使用form提交
  2. 禁止cors（跨域请求）
  3. 检查referrer header，看来自哪个网站
  4. 使用csrftoken，具体的步骤是，服务器发送给客户端一个token，客户端需要提交带有token的表单，接着服务器验证token是否合法，否则拒绝该请求

有个疑问，为什么有些服务器把返回来的csrftoken显示在浏览器上？这样不是很容易被爬虫吗？

先序遍历：

  1. 当栈或是当前节点不为空，则进行下面的循环
  2. 如果当前节点不会空，则访问该节点，并入栈，并将当前节点指向它的左节点，一直循环，直到当前节点为空
  3. 此时说明走到了最左边，则获取栈顶节点，并弹出它，将当前节点设置为它的右节点

对于深度遍历，采用递归的解法很简单，如果使用栈来实现，那么思路是这样的：

  1. 首先将需要遍历的节点放入栈里面，访问，并且设置已经访问
  2. 当栈不为空的时候，按照下面的步骤：
  3. 获取栈顶的节点（不能弹出），并获取它的邻居节点，开始计数，表示已经访问过的邻居节点的个数
  4. 对于每个邻居节点，如果它没有被访问过，则入栈，并访问它，设置已经访问过，并立即退出循环（否则就没有深度往下走了）
  5. 在遍历邻居节点的过程中，如果某个邻居被访问过了，则计数加一
  6. 当计数器的值等于了邻居节点的个数，说明栈顶节点的邻居都访问过了，这时就可以让它出栈了

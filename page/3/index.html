对于深度遍历，采用递归的解法很简单，如果使用栈来实现，那么思路是这样的：

  1. 首先将需要遍历的节点放入栈里面，访问，并且设置已经访问
  2. 当栈不为空的时候，按照下面的步骤：
  3. 获取栈顶的节点（不能弹出），并获取它的邻居节点，开始计数，表示已经访问过的邻居节点的个数
  4. 对于每个邻居节点，如果它没有被访问过，则入栈，并访问它，设置已经访问过，并立即退出循环（否则就没有深度往下走了）
  5. 在遍历邻居节点的过程中，如果某个邻居被访问过了，则计数加一
  6. 当计数器的值等于了邻居节点的个数，说明栈顶节点的邻居都访问过了，这时就可以让它出栈了

关于5种I/O模型的理解

以喝咖啡举例， 在网上点了咖啡粉外卖，要等外卖小哥送咖啡，然后泡咖啡，喝咖啡

  1. 阻塞式I/O：一直站在家里等咖啡（其它什么事不做）送到，然后泡咖啡
  2. 非阻塞式I/O：不停地给外卖小哥打电话（其它什么事不做），送到后泡咖啡
  3. IO多路复用：一开始和小哥说好，到了后打电话，同时站在家里等（于此同时，他还在等其它的外卖，也是什么事不做），一旦咖啡到了，就泡咖啡
  4. 信号驱动式IO：干自己的事情，外卖小哥到了后打电话让你取咖啡，然后你泡咖啡
  5. 异步IO：一直干自己的事情，整个事情都让女朋友来做，做好了通知自己过去喝

mybatis中#和$的区别

#会进行预编译，按照类型匹配的，用于变量替换。

$不进行数据类型匹配，简单的字符串拼接，一般不能用占位符的用它

如果两个都可以用，最好用#

select的时候一定要加resultType或者resultMap，否则报错

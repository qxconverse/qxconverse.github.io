spring的核心是aop，aop的底层实现是代理模式，具体为动态代理。
同时，拦截器是aop的实现之一，它是基于反射的

spring bean的生命周期，这是spring的基础，一定要熟悉。

  1. 首先得初始化，也就是new的过程。通过构造方法或是工厂方法
  2. 设置对象的属性，setter这些，算是依赖注入。
  3. 检查aware相关的接口，并设置相关依赖
  4. 把bean实例传递给后置处理器的postProcessorBeforeInitialization方法
  5. 调用initial-method初始化方法
  6. 把bean实例传递给后置处理器的postProcessorAfterInitialization方法
  7. 使用Bean
  8. IoC容器关闭之前，调用bean的destroy方法

aop的底层原理

  1. 基于JDK的动态代理，只针对实现了接口的业务对象。
  2. CGLib，通过动态地对目标对象进行子类化来实现aop代理。
  3. 业务对象没有实现任何接口的时候默认使用

Spring Bean的scope，这个scope即它的作用域。在Spring中，一共有5种作用域，分别为：

  1. Singleton，单例，这是默认的作用域类型，表示在IoC容器中只会创建一个共享的Bean，即唯一的实例。
  2. Prototype，每次请求，比如将其注入到另外一个Bean，或是在程序中调用getBean的方法时，都会产生一个新的实例。
  3. Request，每一次Http Request都会实例化一个Bean，只在当前的Http Request中有效。
  4. Session，每一次Http Request都会实例化一个Bean，只在当前的Http Session中有效。
  5. Global Session，只在基于portlet的web应用中才有意义，类似于Session的作用域。

进入到项目的CI设置界面，查看下面2个配置参数

  2. Use the following registration token during setup: xxx

注意：

  1. 其中的xxx是需要根据实际情况进行配置的，包括镜像名称、注册token、标签等，其中后两者可在gitlab项目的CI/CD配置中进行配置；其中的docker-image可以使用官方默认的，也可以自己生成镜像以适应不同的需求

ci/cd问题记录

启动报错，发现是无法读取到uc配置，后来看服务器加载的是test1的配置文件，而项目中没有。

于是慢慢找到了setenv.sh文件，里面SPRING_PROFILES_ACTIVE并没有写成test，而是自动根据所选择的所以就改过去了。

不过发现不改也有好处，就是可以配置不同的test环境，在项目中可以指定test1/test2配置文件，而不是仅仅只有一个test配置文件。

部署时如何串起来

1.jenkins里面加上env参数，自动拼接出应用模块名。比如test1

2.根据模块名就会部署到对应的服务器

3.根据当前服务器的应用模块名可以获取到域名

4.根据域名的前缀匹配setenv中的规则，如test1域名前缀，就会去找test1前缀的配置文件

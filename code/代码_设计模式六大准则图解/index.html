设计模式六大准则图解

发表于 2021-01-25 | 更新于: 2021-01-26 | 分类于 代码 | 浏览 次
字数统计: 167 | 阅读时长 ≈ 1

对设计模式6大准则的图解。

依赖倒置原则

核心：面向接口编程

单一职责原则

核心：一个类只负责一项职责

里氏替换原则

核心：子类可以扩展父类的功能，但不能改变父类原有的功能

接口隔离原则

核心：对接口拆分粒度的掌控

迪米特法则

核心：一个对象应该对其他对象保持最少的了解，以尽量降低类与类之间的耦合。

开放闭合原则

此为总则：用抽象构建框架，用实现扩展细节。具体怎么做，就是上面的5点

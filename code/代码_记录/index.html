序列化

对象转化成字节流；serialVersionUID是为了保证反序列化时的一致性。待反序列化对象和目标类的uid要一致才可以。最好加上这个，不加会默认生成，可能会有环境敏感性问题。会被jvm特殊处理。transient关键字的作用是不序列化某个字段。

泛型

目的和思想：todo

泛型擦除：只存在于编译阶段，运行期会擦除。Box<Integer>和Box<Number>都会变成Box。

Box<Integer>和Box<Number>不是父子关系。

类型通配符一般是使用 ? 代替具体的类型实参。Box<?>在逻辑上是Box<Integer>、Box<Number>…等所有Box<具体类型实参>的父类。

使用?太粗暴，怎么细化。涉及到类型通配符上限和类型通配符下限。即只能是某个类的子类或父类。

数据库

数据库行锁和表锁

https://zhuanlan.zhihu.com/p/52678870

问题：

  1. 为什么插入就锁表；而更新只锁行。难道不是都会么？

总结：

  1. 很多地方涉及到事务和隔离级别，再比如spring中的事务等
  2. 并发、保证数据一致性和完整性
  3. 既然有锁，就有钥匙。行锁可以把索引理解为钥匙。表锁是非索引字段，表锁响应的是非索引字段，即全表扫描。
  4. 记录锁，命中唯一索引，如id
  5. 间隙锁（区间锁），命中索引，且没有查询到符合条件的数据。区间是左开右闭
  6. 临键锁，mysql的行锁默认就是使用的临键锁。命中索引，且查询到符合的数据。mysql的索引和B+树相关。这个锁的行为也和它相关。具体情况很复杂。
Mybatis

mybatis-plus如何执行BaseMapper接口里面的方法（没有任何实现）

MybatisMapperProxy代理类的invoke方法；其中会执行MybatisMapperMethod中的execute方法；接着会调用SqlSessionTemplate里面的selectOne方法。它也是通过代理实现的。它里面有个sqlSessionProxy，是一个代理对象，每次调用它的某个方法，会执行SqlSessionInterceptor类里面的invoke方法。具体是多个线程会再去创建不同的sqlsession，具体实现方式是使用TransactionSynchronizationManager类。这样也就是线程安全了。

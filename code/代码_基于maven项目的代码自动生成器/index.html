我们的目的是生成实体类、mapper类和xml文件，这些内容是很有规律的，如下表

文件类型 文件名 字段名 方法名
entity类 表名+Entity.java 表字段 -

也就是说，这些代码都可以通过渲染模板的方式生成，只需要将类名、字段名替换进去即可。因此，要自动生成代码，我们核心要解决的问题是：如何获取表中字段的信息？

借助于现有工具

以java系为例，这是mybatis-plus官网给出的介绍：

AutoGenerator 是 MyBatis-Plus 的代码生成器，通过 AutoGenerator 可以快速生成 Entity、Mapper、Mapper XML、Service、Controller 等各个模块的代码，极大的提升了开发效率。

这是它们核心的生成器类，可以看出主要包括这几个配置：数据源配置、策略配置、包配置、模板配合、全局配置；另外还有一个模板引擎，用来根据模板生成文件的。

假设有现有的工具可以生成代码，我们需要给它提供什么信息呢？

  1. 生成到哪里？
  2. 生成哪个库的哪个表相关的文件？
  3. 生成哪些文件（xml、entity等），其内容、样式怎么控制？
  4. 生成的策略，比如命名规范、用到swagger、lombok时怎么生成？

如何结合到项目中

嵌入式

每个项目新建一个代码自动生成器模块，文件直接生成到对应的模块中。缺点：冗余

项目化

自动生成器项目会生成文件到对应项目的模块中；缺点：项目耦合、生成过程繁杂

插件化

每个项目单独引入代码自动生成器插件，自行维护配置文件，文件直接生成到对应的模块中。

为什么要加索引，其目的是干嘛的？

索引的本质是一种数据结构。最直接的类比对象是字典。索引会将某列的数据放在数据结构中，并且保存指向某行的地址值。

基本原则是只如果表中某列在查询过程中使用的非常频繁，那就在该列上创建索引。

mybatis中#和$的区别

#会进行预编译，按照类型匹配的，用于变量替换。

$不进行数据类型匹配，简单的字符串拼接，一般不能用占位符的用它

如果两个都可以用，最好用#

select的时候一定要加resultType或者resultMap，否则报错
mybatis中判断不为空，且不为空字符串

mybatis中判断不为空，且不为空字符串，适合字符串的变量；如果是整数型变量，当整数不为0时，没问题；但是如果整数为0，那么会判断它的值为空字符串，导致where中不起作用。

Druid是为监控而生的数据库连接池

和java里面线程池、httpclient里面的http请求池类似，凡是涉及到连接、断开操作的东西，都可以有池的概念。为的是更好地实现资源复用。数据库连接池的基本原理是在内部对象池中维护一定数量的数据库连接，并对外暴露数据库连接获取和返回方法。

连接池只应该有maxPoolSize和minPoolSize，druid只保留了maxActive和minIdle，分别相当于maxPoolSize和minPoolSize。

参数 讲解
maxActive 最大连接池数量
minIdle 最小连接池数量
activeCount 当前连接数
connectCount 总的连接次数
closeCount 总的断开连接次数
poolingCount 连接池中的连接数
createCount 创建的连接数
recycleCount 回收的总连接数
activePeak 当前连接数的峰值
poolingPeak 连接池中连接数的峰值

发现很多地方都有idle这个字眼，idle的意思是：懒散的、闲置的。

这是线程池的状态信息：从中可以分析出各个参数是什么意思。

对于连接池的细节，其为一个生产者/消费者模式，使用ReentrantLock + Condition的方式，即锁与同步方式

Condition是为了代替传统的Object的wait()/notify()方法，实现线程间的协作。其核心函数为await()、signal()方法，作用分别对应wait()和notify()。

await：释放锁、沉睡、等待唤醒；signal：获取锁，开始做事，完毕后，调用signal方法，唤醒沉睡的线程。

将锁、countdownLatch、Future类解决超时问题等串起来。

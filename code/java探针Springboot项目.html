Java探针SpringBoot项目

通过本博客，您能完成的目标是

  1. 创建一个javaagent项目，并打包成探针包（agent包）
  2. 监控已有的SpringBoot项目某个类下所有方法的执行时间

需要的环境

  1. Idea开发环境
  2. javassist依赖包
  3. SpringBoot web项目

javaagent项目

创建一个maven项目，其目录结构如下：

核心为MyAgent类和MyTransformer类，第一个类可以理解为agent的入口，需要定义一个premain的方法，方法里面加入一个自定义的transformer类，即MyTransformer，可以理解为写入代码的具体逻辑。

MyTransformer类需要实现ClassFileTransformer类中的transform方法。该方法最后会返回一个字节数组。在本例中的具体逻辑为，在类加载过程中，如果找到了我们需要探针的类UserController时，则进行加入代码操作。否则，返回null，表示不做任何操作。

具体所加入代码的作用是在方法执行之后打印该方法执行的时间。当找到被探针类后，获取其下面所有的方法，接着写入加入的代码，就像我们正常写代码那样。区别在于，传入的是代码字符串，比如beginTime = System.currentTimeMillis();。这样就可以监控UserController类中所有方法的执行耗时了。

附：遇到的坑

在进行javaagent的实践时，一开始mvn package打出的agent包，在执行探针命令后，发现transformer类中的代码执行到ClassPool pool = ClassPool.getDefault();时就没有下文了。最后意识到，可能是因为jar包中没有javassist的依赖，所以找不到ClassPool类。这个问题隐藏很深，因为没有任何报错。后来依据这篇博文https://www.jianshu.com/p/9c284ed187e0 ，在pom文件中加入maven-assembly-plugin， 这样就可以将依赖包javassist打入jar包了。

后来遇到了无法加载类的问题，参考这篇博客解决了：https://www.kagura.me/dev/20190923102501.html

如果遇到"Invalid constant type 18”的问题，则需要替换依赖版本，javassist要改成3.20版本

最后遇到的问题javassist Type top (current frame, locals[2]) is not assignable to long，参考了这篇博客解决了，原来是加入的代码问题：https://stackoverflow.com/questions/34223471/how-to-get-arround-java-lang-verrifyerror
